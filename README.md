ОТЧЁТ ПО ПРАКТИКЕ: Инструмент визуализации графа зависимостей для менеджера пакетов

ВАРИАНТ 10

Цель практики: разработка CLI-приложения, способного строить граф зависимостей пакетов, 
с поддержкой конфигурации, тестирования и анализа с учётом максимальной глубины и циклов.

========================================================================================

Этап 1. Минимальный прототип с конфигурацией

Цель: создать минимальное CLI-приложение и сделать его настраиваемым.

Реализация:
- Приложение читает параметры из YAML-файла.
- Поддерживаются настраиваемые параметры: имя пакета, URL/путь к репозиторию, 
  режим работы (test/repo), имя выходного файла изображения, максимальная глубина анализа.
- При запуске выводятся все параметры в формате ключ-значение.
- Реализована валидация и обработка ошибок для всех параметров.
- Используется библиотека PyYAML для парсинга конфига.

========================================================================================

Этап 2. Сбор данных

Цель: реализовать основную логику получения данных о зависимостях.

Реализация:
- Используется формат пакетов Ubuntu (apt).
- Приложение скачивает файл Packages или Packages.gz с указанного URL.
- Извлекаются прямые зависимости указанного пользователем пакета.
- Выводятся все прямые зависимости на экран.
- Запрещено использовать готовые инструменты типа apt-cache.
- Используются только встроенные библиотеки: urllib, gzip, io.

========================================================================================

Этап 3. Основные операции

Цель: построить граф зависимостей (с учётом транзитивности) и выполнить 
      основные операции над ним.

Реализация:
- Граф зависимостей строится с помощью алгоритма BFS (поиск в ширину) без рекурсии.
- Учитывается максимальная глубина, заданная пользователем.
- Обрабатываются циклические зависимости — пакеты не посещаются дважды.
- Поддерживается тестовый режим: вместо URL указывается путь к файлу с описанием графа.
  Пример формата: A: B, C
                  B: D
                  C: D, E
- Демонстрация работы проводилась на различных случаях: циклы, ограниченная глубина.

========================================================================================

Заключение:
Все три этапа успешно реализованы. Приложение работает как в режиме реального 
репозитория (через URL), так и в тестовом режиме (через файл). Поддерживает 
валидацию, обработку ошибок, корректную работу с циклами и ограничением глубины.
